# Cidre v0.20.0 Controlled Boot Test Pack

This document covers the implementation plan, checklists, and details for the Cidre v0.20.0 release.

## 1. 目的

Cidre v0.20.0 の目的は、v0.19.0 で生成・登録・検証できるようになった Cidre prototype image を、実機 Apple Silicon Mac 上で controlled boot test し、firstboot OOBE / seed resume / handoff / recovery log collection の実動作を確認できるようにすることである。

## 2. 実装スコープ

- **Preflight Checks (`cidre-boot-preflight`)**: テスト前に入力となるイメージの存在、ハッシュ合致、およびメタデータの整合性を検査する。
- **Controlled Orchestration (`cidre-controlled-boot-test`)**: テストフローの準備、ログ回収、レポート出力を一括コントロールする。
- **Observation Logging (`cidre-boot-observe`)**: 人間が観察したコンソール表示や OOBE 状態を構造化した Markdown ファイルに記録する。
- **State Validation (`cidre-firstboot-verify`)**: 起動後に作成される起動マーカー（started / completed / failed）の有無や handoff.txt 内容を検証する。
- **Results Recording (`cidre-boot-result` / `cidre-boot-failure-classify`)**: 成否ステータスを JSON 記録し、エラー時は既知の 10 カテゴリに分類する。
- **Status Dashboard (`cidre-doctor` / `cidre-recovery`)**: `--controlled-boot` および `boot-test-status` でログ回収状況や直近のテスト成否をダッシュボード化する。

## 3. 非目標

- パブリックブートイメージのリリース
- Apple Silicon ブートポリシーの全自動書換え
- macOS 側のインストーラー完全自動制御
