# v0.35.6 Controlled Manual-Boot Install

## Summary

`v0.35.6` adds the first controlled manual-boot install pack for the macOS
installer flow.

The release is aimed at making real-machine validation safer by separating:

- target selection
- payload staging
- boot choice
- post-run evidence review

## Added

- controlled install planning
- install target validation
- payload metadata and verification tooling
- manual boot guide generation
- no-default-boot verification support
- SwiftUI views for the controlled install path

## Safety Position

This release does not re-enable normal destructive install automation.

It is still governed by the containment posture introduced after
`DFU_RESTORE_001`:

- disk-changing install remains disabled by default
- finish/restart/shutdown remain gated
- protected Apple partition changes remain unacceptable
- default boot mutation remains out of scope

## Expected Usage

The intended usage model is:

1. validate the target
2. prepare the payload
3. review the manual boot instructions
4. choose the boot target manually
5. verify that macOS boot behavior did not regress

## Exit Criteria Before Wider Use

Before this release can be treated as more than a controlled test vehicle, the
project still needs:

- disposable-target live drill evidence
- repeatable no-default-boot verification on real machines
- documented first-boot success results
- documented macOS return-path success results
- confidence that the finish/shutdown gates correctly prevent unsafe completion
