# Cidre.app Terminal-Free Policy

The normal install path stays inside `Cidre.app`. The app scans disks, creates and previews plans, requests macOS administrator authentication, executes the bundled helper, records results, and presents failures without asking the user to open Terminal.

Developer scripts remain available for diagnostics and automated validation, but they are not required by the setup wizard. The packaged app installs its backend into Application Support on first launch.
