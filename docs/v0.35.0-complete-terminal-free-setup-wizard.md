# Cidre v0.35.0: Complete Terminal-Free Setup Wizard

Cidre v0.35.0 packages a launchable macOS app at `dist/Cidre.app`. On first launch, the app installs its bundled backend into `~/Library/Application Support/Cidre/Backend` and opens the linear setup wizard.

Disk changes use this sequence: scan, plan, preview, exact confirmation, macOS administrator authentication, helper validation, `diskutil` execution, and audit logging. The helper accepts only explicitly supported operations and passes arguments directly to `Process`; it never evaluates user input through a shell.

The setup Disk Plan screen detects the startup APFS physical store, accepts the desired post-resize container size and new partition size, and previews the exact `diskutil apfs resizeContainer` invocation. No change occurs until the user types the generated `CIDRE EXECUTE <plan-id>` phrase and approves macOS authentication.

Local packaging uses ad-hoc signing. Public distribution still needs Developer ID signing and notarization credentials.
